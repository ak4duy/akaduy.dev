8:T10d0,# Rust and First Impressions

Rust is a pretty interesting language, and also a pain for a lot of people.

At first, I found Rust the same way most programmers do: through jokes about the borrow checker, the "strict" compiler, and memes about fighting Ownership at 3 AM. Back then, I didn't really understand why people both loved and hated it so much.

![](https://i.redd.it/x3at51cabj7a1.png)

I used to think:

> "Why make things so complicated when Java or Python make them much easier?"

When writing Java, I didn't have to think too much about where data lived, who owned it, or when it would be freed. Python felt even more relaxed. Write code, run it, fix bugs if they appear.

Rust felt different.

It felt like having an extremely strict roommate. Make one mistake and you immediately get called out. There were times when I just wanted to write something simple, only for the compiler to throw an entire wall of errors at me and make me question whether I was fighting my own code or the computer itself.

But the longer I used Rust, the more I realized the compiler wasn't trying to make my life harder.

It was trying to stop me from doing something stupid ><.

# What Is Rust Trying to Solve?

Modern software keeps getting more complicated. People want programs to run faster, stay more stable, make better use of hardware resources, and have fewer bugs. But the closer we get to low-level performance, the more painful problems begin to appear.

Some bugs don't show up immediately. Some only appear after hours of runtime. Some bugs only need one small mistake to turn an entire evening into a desperate bug-hunting session.

Rust was built to solve a huge portion of those problems.

# What Makes Rust Interesting to Me

Also, what makes Rust exciting to me is not just one thing.

I like that it is fast. I like that it tries to solve memory safety problems at compile time. I like its approach to concurrency. And I also like seeing more major companies start using Rust in real-world systems.

There is something exciting about seeing a relatively new language slowly appear in more places. The Linux kernel started supporting Rust. Large technology companies are investing in it. More libraries and tools continue to grow around its ecosystem.

# Ownership Changed How I Think

And the thing that changed me the most was Rust's incredibly consistent way of handling Ownership.

At first... I honestly hated it. I could not understand why the compiler would not let me do things that felt completely normal. 

Why did this variable get moved? Why does this borrow fail? Why does changing one line suddenly create a whole bunch of other errors?

But after some time, I realized Rust was forcing me to think more carefully.

Who owns this data? Who is allowed to modify it? Does this data still exist at this point?

Those questions may sound small. But they changed the way I write code, not only in Rust, but in other languages too.

# Cargo and the Ecosystem Are Amazing

Another thing I absolutely love is Cargo.

Some technologies just feel like everything fits together naturally. Cargo gives me that feeling. Managing dependencies, building projects, formatting code, running tests, everything feels clean and unified.

The ecosystem feels designed to work together instead of feeling stitched together afterward.

# Is Rust the Future?

Could Rust become the language of the future?

I think it could. Maybe even replace a lot of things, though that still depends heavily on the field.

Python will remain strong.

Java will continue existing, even though everyone jokes about hating it.

C++ isn't disappearing overnight.

But I think Rust is moving in the right direction.

In a world where software increasingly needs higher performance, stronger reliability, and better resource usage, I think the ideas Rust is pushing toward will only become more important.

Maybe the future isn't "everyone uses Rust."

But I think the future will definitely include Rust.

Or at least, it's a future I want to keep learning and exploring. And if you ask me what language someone should start learning... Well, you probably already know my answer: Rust 🦀
